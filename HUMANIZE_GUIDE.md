# AI 写作去 AI 化指南

## 📋 核心原则

**最重要的一点**：AI 写作的目标是"真实"，不是"完美"。人类写作有温度、有瑕疵、有个性，这才是最好的伪装。

---

## 🎯 AI 写作的典型特征（需要避免）

### 1. 句式结构问题
- ❌ **过度使用复杂句**：Because it was raining, the event was postponed.
- ❌ **句子长度过于一致**：每段都是 3-4 句，每句都差不多长
- ❌ **机械的节奏感**：所有段落结构完全一样

### 2. 用词问题
- ❌ **过度正式**："进行"、"实施"、"开展"、"It is important to note"
- ❌ **情感词汇贫乏**：被 RLHF 训练得过于"无害"，缺少强烈情感
- ❌ **重复使用相同的过渡词**："首先、其次、最后"、"In light of this"

### 3. 标点符号问题
- ❌ **滥用破折号作为结论**：This is the key point—like this.
- ❌ **标题全部首字母大写**：Every Word In The Title Is Capitalized

### 4. 内容问题
- ❌ **开头太正式**："在当今社会"、"随着时代发展"、"In today's world"
- ❌ **结尾太升华**："让我们一起"、"相信未来"
- ❌ **空洞的真理**：没有实质内容的陈述（vacuous truths）
- ❌ **缺少冲突和矛盾**：所有论证都太完美、太顺畅

## ✅ 去 AI 化的 10 个实战技巧

### 技巧 1：打破句式结构的规律性

**AI 版**：
```
时间管理很重要。我们可以用番茄工作法提高效率。这个方法很有效。
```

**人类版**：
```
说到时间管理，我以前也是个拖延症重度患者。

后来试了好多方法，什么 GTD、四象限，都没啥用。直到遇到番茄工作法，才算找到点感觉。
```

**要点**：
- 混合使用短句、中句、长句
- 单句成段制造停顿感
- 避免每段都是 3-4 句的固定模式

---

### 技巧 2：加入个人声音和视角

**AI 版**：
```
读书可以增长知识，开阔视野。
```

**人类版**：
```
读书这事儿吧，说实话我以前挺抗拒的。

但有次在地铁上，看到一个大叔捧着本厚书，那专注的样子突然打动了我。后来我也开始读书，发现还挺上瘾。
```

**要点**：
- 使用第一人称："我觉得"、"说实话"、"坦白讲"
- 加入个人经历和观察
- 使用口语化表达

---

### 技巧 3：加入真实细节

**AI 版**：
```
她很努力工作，最终获得了成功。
```

**人类版**：
```
小李每天早上 6 点就到公司，晚上 11 点才离开。

去年 10 月的那个项目，她连续加班了 3 周，最后在星巴克的角落里，对着电脑哭了一场。但项目成功了，她也升职了。
```

**要点**：
- 具体的时间、地点、人名
- 细节描写（场景、对话、动作）
- 避免泛泛而谈

---

### 技巧 4：使用情感化语言

**AI 版**：
```
这个方法很有效。
```

**人类版**：
```
这个方法真的救了我。说实话，当时我都快崩溃了。
```

**要点**：
- 使用强烈的情感词：崩溃、兴奋、沮丧、惊喜
- 避免过度克制和理性
- 加入语气词："真的"、"说实话"、"哎呀"

---

### 技巧 5：制造"不完美"

**AI 版**：
```
坚持很重要，但也要注意方法。
```

**人类版**：
```
坚持很重要吗？当然重要。

但光坚持就够了吗？好像也不是。你得找对方法，不然就是瞎坚持。

不过话说回来，有时候瞎坚持也比不坚持强。这事儿挺矛盾的。
```

**要点**：
- 自问自答
- 承认矛盾和不确定性
- 不把话说满

---

### 技巧 6：加入"废话"和铺垫

**AI 版**：
```
番茄工作法可以提高效率。
```

**人类版**：
```
说到提高效率，我得先吐槽一下。

以前我试过各种方法，什么 GTD、四象限、时间块，听起来都挺厉害的，但实际用起来，emmm，怎么说呢，就是不太适合我。

直到后来，我朋友推荐了番茄工作法。一开始我还挺怀疑的，心想这么简单的方法能有用吗？结果试了一下，嘿，还真行。
```

**要点**：
- 加入铺垫和背景
- 允许跑题和闲聊
- 不要每句话都直奔主题

---

### 技巧 7：变化过渡方式

**AI 版**：
```
首先，我们需要...
其次，我们应该...
最后，我们要...
```

**人类版**：
```
先说第一点。

说到这个，你可能会问...

对了，还有一个很重要的事...
```

**要点**：
- 避免"首先、其次、最后"
- 使用口语化过渡："说到这个"、"对了"、"顺便说一句"
- 用问句引导："你可能会问"、"那怎么办呢"

---

### 技巧 8：控制破折号的使用

**AI 版**：
```
这是最重要的一点—坚持。
成功的关键—努力。
```

**人类版**：
```
这是最重要的一点：坚持。

成功的关键？努力。当然，还有运气。
```

**要点**：
- 破折号用于插入语，不用于结论
- 结论用冒号、问号、句号
- 避免每段都用破折号

---

### 技巧 9：加入冲突和矛盾

**AI 版**：
```
这个方法很好，大家都应该尝试。
```

**人类版**：
```
这个方法挺好的，但也不是万能的。

有些人用了效果很好，有些人用了反而更乱。我自己用了半年，有时候觉得很有用，有时候又觉得是在浪费时间。

说实话，我也不知道该不该推荐给你。
```

**要点**：
- 承认局限性
- 展示不同观点
- 避免一边倒的论证

---

### 技巧 10：使用第二人称互动

**AI 版**：
```
人们应该注意时间管理。
```

**人类版**：
```
你有没有这种感觉？

每天忙得要死，但回头一看，好像什么都没干成。我以前就是这样，直到我开始注意时间管理。
```

**要点**：
- 使用"你"、"我们"而不是"人们"
- 提出反问句
- 制造互动感

## 🎨 不同文章类型的去 AI 化策略

### 鸡汤励志文
- ✅ 多用排比，但不要太工整
- ✅ 金句要接地气，不要太高大上
- ✅ 加入生活场景，不要只讲道理
- ✅ 开头和结尾可以用名人名言，但中间要接地气

### 职场干货文
- ✅ 加入个人经验："我试过"、"我踩过的坑"
- ✅ 承认局限性："这个方法不一定适合所有人"
- ✅ 用第一人称，不要总是"我们"
- ✅ 加入具体案例和数据

### 情感故事文
- ✅ 细节要具体，不要泛泛而谈
- ✅ 对话要自然，不要像剧本
- ✅ 情节要有波折，不要太顺
- ✅ 情绪要真实，不要过度煽情

## 🔧 实用工具和方法

### 方法 1：大声朗读测试
写完后，大声读一遍。如果读起来不像人说话，那就是 AI 味太重。

### 方法 2：逐句检查清单
- [ ] 这句话我会这样说吗？
- [ ] 这个词太书面了吗？
- [ ] 这个过渡太生硬了吗？
- [ ] 这段话有个人特色吗？

### 方法 3：AI 辅助去 AI 化
在 prompt 中加入去 AI 化指令：

```
在写作时，请注意：
1. 避免过度正式的表达
2. 使用口语化的语言
3. 加入个人视角和经历
4. 变化句式结构，避免机械感
5. 加入情感和细节
6. 不要把话说得太满
7. 允许矛盾和不确定性
```

## ⚠️ AI 痕迹快速排查表

### 高频 AI 套话（发现即删）

| 类别 | ❌ 必须删除 | ✅ 替换方案 |
| --- | --- | --- |
| 开头套话 | 在当今时代 / 随着...的发展 / 在...的背景下 | 直接切入故事或场景 |
| 过渡套话 | 首先 / 其次 / 最后 / 综上所述 / 总而言之 | 说到这个 / 对了 / 顺便说一句 |
| 强调套话 | 值得注意的是 / 需要指出的是 / 毫无疑问 | 注意 / 重点来了 / [直接说内容] |
| 结论套话 | 让我们一起 / 相信未来 / 相信大家 | [直接给结论，不升华] |
| 书面词 | 进行 / 实施 / 开展 / 显著提升 / 充分利用 / 深入了解 | 做 / 用 / 开始 / 涨了 X 倍 / 用好 / 搞清楚 |
| 判断句式 | A 是 B（定义句）/ 所谓 A 即 B | 改成描述行为或感受：「用起来像」「感觉是」 |

### 过度句式（严格控制）

| ❌ 过度使用 | ✅ 改法 |
| --- | --- |
| 「A 是错的，B 才是对的，不是 C 还是 D」 | 直接说 B，整篇最多用 1 次 |
| 「既...又...不仅...而且...同时...」 | 拆成多个独立短句 |
| 「从 A 到 B，从 B 到 C」排比 | 直接列举 A、B、C |
| 「非常 / 极其 / 十分 + 形容词」 | 删除副词，换成具体描述 |

### 中英文混排规范（公众号必须遵守）

公众号在手机上阅读，混排不规范会非常显眼。**写作时必须遵守以下 4 条规则：**

1. 中文 + **空格** + 英文 / 数字
2. 英文 / 数字 + **空格** + 中文
3. 英文 / 数字 + 标点（**无空格**）+ 中文
4. 中文 + 标点（**无空格**）+ 英文 / 数字

**正确示例：**
- 「我用 Cursor 做了一个项目」
- 「AI 功能使用量涨了 10 倍」
- 「获得了 5000 美金（约 3.5 万人民币）」
- 『这是「AI 友好」的文档』

**错误示例：**
- ❌ 「我用Cursor做了一个项目」（缺空格）
- ❌ 「我用 Cursor 、 Claude Code」（标点前多余空格）
- ❌ 「获得了 5000 美金,约 3.5 万人民币」（英文逗号）

### 口语表达模式速查

| 场景 | ❌ 书面词 | ✅ 口语替换 |
| --- | --- | --- |
| 开头 | 在当今时代 / 随着科技发展 | 直接场景切入：「说起来...」/「你有没有这种感觉...」|
| 转折 | 然而 / 但是需要指出的是 | 但是 / 问题是 / 重点来了 |
| 强调 | 值得注意的是 / 这一点至关重要 | 注意 / 说实话 / 核心是 |
| 总结 | 综上所述 / 总而言之 | 说白了 / 简单来说 / [直接给结论] |
| 自问 | 为什么？ | 就是因为... / 原因很简单... |
| 不确定 | 可能会 / 可能有效 | 我用下来感觉... / 我也不确定 |

## 📊 替换词限频元规则（重要）

删 AI 套话后，最容易掉进的新坑是同一个口语词满篇飞——「说白了」说 5 遍反而是新的 AI 味。

**单文限频（硬性）：**
- 「说白了」整篇 ≤ 1 次
- 「为什么？」整篇 ≤ 1 次（优先用"就是因为...）
- 「重点来了」整篇 ≤ 1 次
- 「很清楚 / 非常清楚」直接删除，不替换

## 📊 去 AI 化效果评估

### 自我检查清单
- [ ] 句子长度有变化吗？
- [ ] 有个人视角和经历吗？
- [ ] 有具体细节吗？
- [ ] 有情感表达吗？
- [ ] 有"不完美"的地方吗？
- [ ] 读起来像人说话吗？

### 测试方法
1. **朗读测试**：大声读一遍，听起来自然吗？
2. **朋友测试**：让朋友读一遍，问他们觉得是人写的还是 AI 写的
3. **AI 检测工具**：用 GPTZero、Originality.ai 等工具检测

## 💡 终极秘诀

**记住**：
- AI 写作是框架，人类编辑是灵魂
- 不完美，才是最好的伪装
- 真实的情感和细节，是 AI 无法模仿的
- 去 AI 化是为了让内容更有温度，不是欺骗读者

**工作流程**：
1. AI 生成初稿（框架）
2. 人类编辑（加入灵魂）
3. 去 AI 化处理（打磨细节）
4. 最终审核（确保自然）

## 🎯 在 wechat-article-publisher 中的应用

创作时参照 SKILL.md 的**三遍审校流程**（`references/三遍审校清单.md`）：
- 第一遍 内容审校：事实 / 逻辑 / 结构
- 第二遍 风格审校：用本指南的替换词表 + 限频规则
- 第三遍 细节打磨：句长 ≤ 30 字 / 全角标点 / 中英文混排空格

朗读测试未通过的文章不得进入下一步（生成封面 / 创建草稿）。
